# Security Policy

## Reporting a vulnerability

Si encuentras una vulnerabilidad de seguridad, por favor **no abras un issue público**.

Usa uno de estos canales:

- GitHub Security Advisories (recomendado): ve a la pestaña **Security** del repositorio y crea un **Private vulnerability report**.

- Alternativa (si no está habilitado en tu repo): contacta al mantenedor por un canal privado y comparte descripción del impacto, pasos para reproducir, versiones afectadas y un PoC (si aplica).

## Supported versions

Este proyecto se despliega como un Cloudflare Worker; se considera “soportada” la versión actualmente desplegada en `main`.

## Response targets

Intentaremos responder en un plazo razonable, pero no se garantizan SLAs.
